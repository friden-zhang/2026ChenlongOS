## 1. SPI 协议是什么
SPI(Serial Peropheral interface, 串行外设接口)是一种高速的、全双工、同步的通信总线，被广泛应用于微控制器(MCU)与各种外围设备(如传感器、Falsh等)之间的短距离通信。


## 2. SPI协议详解
### 2.1 硬件与引脚

SPI 通常需要 4 根线来进行通信（如果是单向传输也可以精简为 3 根线）。它采用主从模式（Master-Slave），通信总是由主设备发起。
这四根线分别是：
- SCLK / SCK (Serial Clock): 串行时钟线。由主设备（Master）产生，用于同步数据传输。
- MOSI (Master Out Slave In): 主设备输出 / 从设备输入线。数据从主设备发送到从设备。
- MISO (Master In Slave Out): 主设备输入 / 从设备输出线。数据从从设备发送到主设备。
- CS / SS (Chip Select / Slave Select): 片选信号线。由主设备控制，通常是低电平有效。当主设备将这条线拉低时，表示选中了该从设备，通信开始。通过提供多个 CS 引脚，一个主设备可以连接多个从设备。

### 2.2 数据传输原理
SPI 的本质是移位寄存器（Shift Register）的环形连接。
主设备和从设备内部各有一个移位寄存器。在 SCLK 时钟信号的驱动下，主设备的寄存器内容和从设备的寄存器内容相当于连成了一个环。
每产生一个时钟周期，主设备将自己寄存器的最高位（或最低位）通过 MOSI 发送给从设备，同时从设备也将自己寄存器的位通过 MISO 发送给主设备。经过 8 个时钟周期后，主从设备正好完成了一个字节的数据交换。因此，SPI 的发送和接收是同时进行的（全双工）。

### 2.3 SPI 的四种工作模式
不同的设备可能在不同的时钟状态下采集和改变数据。SPI 定义了两个配置参数：
- CPOL (Clock Polarity, 时钟极性): 决定 SCLK 在空闲状态（没有数据传输时）是高电平还是低电平。
- - CPOL = 0：空闲时 SCLK 为低电平。
- - CPOL = 1：空闲时 SCLK 为高电平。

- CPHA (Clock Phase, 时钟相位): 决定数据是在时钟的第一个跳变沿（上升沿或下降沿）采样，还是在第二个跳变沿采样。
- - CPHA = 0：在时钟的第一个跳变沿进行数据采样。
- - CPHA = 1：在时钟的第二个跳变沿进行数据采样。

这两个参数组合起来构成了 SPI 的四种模式（Mode 0 ~ Mode 3）。最常用的是 Mode 0（CPOL=0, CPHA=0）和 Mode 3（CPOL=1, CPHA=1）。 通信前，主从设备的模式必须设置一致。

### 2.4 优缺点
优点: 传输速度快（通常可达十几甚至几十 MHz）；全双工通信；硬件协议简单，无需像 I2C 那样发送设备地址。
缺点: 没有应答机制（不知道对方是否真的收到了）；引脚相对较多（每个额外的从设备都需要多一根 CS 线）；只适合板级短距离通信。


## 3. 仿真
> 使用 Wokwi/PlatformIO 进行仿真，支持 VSCode 插件安装，通过 [视频](https://www.youtube.com/watch?v=ECNTyMm_5PE)、[文档](https://docs.wokwi.com/vscode/getting-started) 了解Wokwi基本使用方式

``` bash
cd lecture-0/spi-sim
pio project init --board uno
```

使用 `Arduino Uno` 作为 SPI 主设备，驱动一块 `ILI9341` TFT 屏幕。

### 3.1 接线

这个仿真里实际用到的线并不多，核心就是把 Uno 的硬件 SPI 引脚接到屏幕上：

- `5V -> VCC`
- `GND -> GND`
- `D10 -> CS`
- `D9 -> D/C`
- `D11 -> MOSI`
- `D13 -> SCK`

这里我没有接 `MISO`，因为这个实验只需要把图形数据从 `Uno` 发到屏幕，不需要从屏幕把数据再读回来。

![](https://my-obsidian-vault-1419031144.cos.ap-beijing.myqcloud.com/images/20260418234548120.png)

上面这张图就是接线完成后的样子。可以看到，`D10 / D11 / D13` 这几个和 SPI 相关的数字引脚都用上了，其中：

- `D10` 负责片选 `CS`
- `D11` 是 `MOSI`
- `D13` 是时钟 `SCK`
- `D9` 虽然不是 SPI 硬件专用脚，但这里作为屏幕的 `D/C` 控制脚使用，用来区分“当前发过去的是命令还是显示数据”

### 3.2 代码做的事情

`lecture-0/spi-sim/src/main.cpp` 里主要做了下面几件事：

1. 初始化 `Adafruit_ILI9341` 驱动对象。
2. 把屏幕旋转成横屏。
3. 先画一套静态界面，包括标题栏、接线说明和显示区域。
4. 在 `loop()` 里周期性刷新进度条、颜色条、帧计数和运行时间。

这个程序本质上是在不断调用图形库，把绘图命令和像素数据通过 SPI 一直写给 `ILI9341` 控制器。

### 3.3 运行结果

![](https://my-obsidian-vault-1419031144.cos.ap-beijing.myqcloud.com/images/20260418234606994.png)

上图是仿真运行起来之后的效果。左边是 `Uno`，右边是已经被驱动起来的屏幕。屏幕上可以看到：

- 顶部标题 `ChenlongOS SPI Demo`
- 左侧接线说明
- 右侧动态进度条
- 颜色块
- 帧计数和运行时间

这些内容能持续变化，说明 `Uno` 确实在不断地通过 SPI 向屏幕写数据，而屏幕控制器也在按收到的数据刷新显示。

如果只看这个实验，我觉得它比较直观地说明了 SPI 在实际项目里的一个常见用法：主控并不是只和传感器通信，也经常会通过 SPI 去驱动显示屏、Flash、SD 卡这类外设。这里的 `ILI9341` 屏幕，就是一个很典型的 SPI 外设例子。
